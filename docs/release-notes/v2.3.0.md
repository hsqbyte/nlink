# nlink v2.3.0 — Batch 5: user-visible features

Follow-up to the Batch 1-4 audit rollout (`v2.2.0`). This release ships the four
user-visible items selected from the post-audit roadmap.

Commit: `198258f`

## Highlights

- ✏️ **Proxy edit** — edit an existing proxy from the dashboard, no more delete + re-add.
- 🔁 **SIGHUP hot-reload** — rotate `token` / `token_prev` without restarting the process.
- 🛡️ **VPN policy UI** — tweak per-peer `routes` / `allow_cidr` / `deny_cidr` from the dashboard.
- 🧪 **e2e in CI** — the Docker-based end-to-end smoke test now runs on every push / PR.

## Details

### Proxy edit

- New API: `PUT /api/v1/peers/:name/proxies/:proxyName`
- Dashboard: proxy cards gain a ✎ button that pre-fills the add-proxy form (name disabled, submit button relabelled "更新").
- Gateway path (when the target peer is managed by a remote node) falls back to sequential `remove_proxy` + `add_proxy` forwarded calls — no new TCP command required.

### SIGHUP hot-reload

- `kill -HUP <pid>` re-reads the YAML config file.
- Applied at runtime: `node.token`, `node.token_prev` (useful for zero-downtime token rotation).
- Structural changes (listen port, `vpn.enabled`, peers topology) are logged as "需要重启" and ignored until the next full restart.
- Implementation: `config.ReloadConfig(path)` returns a fresh parsed struct without mutating globals; `config.ApplyReload(newCfg)` performs the scoped mutation.

### VPN policy UI

- New API: `PUT /api/v1/vpn/peers/:vip/policy` with body `{endpoint?, routes[], allow_cidr[], deny_cidr[]}`. If `endpoint` is omitted, the current endpoint is reused.
- Dashboard: VPN peer rows now show `virtual_ip (endpoint) routes=N rtt=Xms ↓bytes ↑bytes` plus an "编辑策略" button that prompts for the three lists.
- Under the hood reuses `UDPTransport.AddPeerWithOptions`, which updates endpoint + Routes/AllowNets/DenyNets in place while preserving atomic metrics.

### e2e in CI

- New `e2e` job in `.github/workflows/ci.yml`, runs on `ubuntu-latest`, depends on `test`.
- Executes `bash deploy/e2e/run.sh`, which builds the Docker image, stands up `nlink-server` + echo + `nlink-client`, waits for `/health`, and round-trips a payload through the tunnel.

## Upgrade notes

- **Breaking**: none.
- **Config additions**: none; all changes are API / UI / runtime.
- **Ops**: to hot-rotate the node token, update the YAML and send `SIGHUP`; other fields still require a restart.

## Verifying locally

```bash
# build + tests
go vet ./... && go test -race ./...

# e2e (same as CI)
bash deploy/e2e/run.sh

# SIGHUP smoke
./bin/nlink -c config/nlink.yaml &
# edit config/nlink.yaml (rotate node.token, keep old value in node.token_prev)
kill -HUP $(pgrep -f 'bin/nlink')
```
